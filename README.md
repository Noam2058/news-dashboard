# מהדורה — דאשבורד חדשות אישי

פרויקט מורכב משלושה חלקים שרצים יחד:

```
news-dashboard/
├── telegram/    → הליסנר של טלגרם (Python) - שולף הודעות מהערוצים
├── backend/     → שרת Node שמריץ RSS + קורא את הודעות הטלגרם + API + עדכונים חיים
└── frontend/    → הדאשבורד עצמו (React), שמציג הכל ומתעדכן בזמן אמת
```

יש להריץ את שלושת החלקים **במקביל**, בשלושה טרמינלים נפרדים.

## 1. הליסנר של טלגרם

```bash
cd telegram
pip install -r requirements.txt
python telegram_listener.py
```
בהרצה ראשונה תתבקש להזין קוד אימות שיישלח לאפליקציית הטלגרם שלך.
מכאן הוא ירוץ ברקע ויכתוב כל הודעה חדשה מהערוצים שהוגדרו (`newsisrael`, `abualiexpress`)
לקובץ `telegram/messages.jsonl`.

## 2. שרת ה-Backend

```bash
cd backend
npm install
npm start
```
השרת יעלה על **http://localhost:4000**. הוא עושה שני דברים ברקע:
- מושך כל 5 דקות כתבות חדשות מ-RSS של גלובס
- עוקב אחרי `telegram/messages.jsonl` וקולט הודעות חדשות תוך כמה שניות

כל הנתונים נשמרים ב-`backend/news.db` (קובץ SQLite), כך שההיסטוריה נשמרת גם
אחרי שסוגרים ומריצים מחדש.

## 3. הדאשבורד (Frontend)

```bash
cd frontend
npm install
npm run dev
```
ייפתח על **http://localhost:5173**. הדאשבורד טוען את ההיסטוריה, ואז נשאר
מחובר לשרת (SSE) ומקבל כל עדכון חדש בזמן אמת — כרטיס חדש מהבהב בזהב לכמה
שניות כשמגיע.

לחיצה על "הפעל התראות" תבקש הרשאת דפדפן, ומאותו רגע כל עדכון חדש גם יופיע
כהתראת מערכת (Notification) — **כל עוד הטאב פתוח**, גם אם הוא ברקע/ממוזער.

## מגבלות ידועות

- **כלכליסט:** לא נמצאה כתובת RSS רשמית ופעילה של כלכליסט נכון להיום.
  המקור מוגדר ב-`backend/src/rssPoller.js` אבל מושבת (`enabled: false`).
  אם תמצא/י כתובת פעילה, אפשר להוסיף אותה שם ולהפעיל.
- **התראות אמיתיות כשהדפדפן סגור לגמרי** (Web Push עם Service Worker) דורשות
  הרצה על שרת עם כתובת ציבורית (HTTPS) ולא רק על המחשב האישי. הגרסה הנוכחית
  נותנת התראות "בזמן אמת" כל עוד הטאב פתוח - זו הדרך הפשוטה ביותר להתחיל.
  אפשר להרחיב לזה בהמשך אם תרצה להריץ את זה על שרת אמיתי (VPS/Raspberry Pi).
- כרגע הכל רץ מקומית (`localhost`) — לשימוש אישי בלבד, מהמחשב שלך.
